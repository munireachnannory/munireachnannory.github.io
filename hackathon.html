⚡ Day 2 · 2026

MFM Combinatorial Library: AI-Powered Magnetic Materials Characterization

📅 March 3, 2026 ⏰ ~3 hours 📉 23 sites · 7,165 particles detected 🏭 Combinatorial materials library
Context

The challenge

Given a real experimental dataset from a 23-site combinatorial magnetic thin-film library — raw AFM/MFM binary files — extract physically meaningful structural and magnetic properties, map them spatially, and deliver insight in roughly 3 hours. No labeled answers. No pre-built pipeline. Just the data, Python, and the clock.

🔮
What makes this a real problem: Combinatorial libraries are used in materials discovery to screen many compositions at once. Rapidly mapping how magnetic properties vary across the library is essential for identifying promising compositions — but the analysis pipeline doesn't come packaged. You have to build it from scratch, fast.
Dataset

What I was working with

The dataset consisted of MFM (Magnetic Force Microscopy) and AFM (Atomic Force Microscopy) images acquired at 23 spatial positions across a combinatorial thin-film sample. Each site contained 5 image channels including height (topography) and MFM phase (magnetic response), at 256×256 pixel resolution.

Approach

Pipeline architecture

I built the analysis in modular stages, each producing a spatial map across the 23 sites. The strategy was to compute per-site scalar metrics, then use cubic interpolation to produce continuous spatial maps — overlaid with the measured site positions to show ground truth alongside the interpolation.

🎵

FFT Autocorrelation — Correlation Length

Wiener–Khinchin theorem applied to MFM phase images. Hanning window to suppress wrap-around artifacts. Radial-averaged autocorrelation decays to find the characteristic magnetic domain size.

⚙

AFM Surface Roughness — Sa, Sq, Rz, Rsk, Rku

Best-fit plane leveling on height channel. Five ISO-style roughness metrics computed per site. Spatial maps reveal how surface texture changes across the combinatorial library.

🔮

Watershed Particle Detection

Otsu thresholding → binary mask → distance transform → peak-local-max markers → watershed labels → regionprops. Applied to all 23 sites. Extracted size, area, perimeter, eccentricity per particle.

📈

Gabor Filter Texture Analysis

Multi-frequency, multi-orientation Gabor filter bank to characterize texture energy, dominant orientation, and anisotropy. Reveals whether the surface has preferential structural directionality.

⚖

Edge Density & Boundary Roughness

Canny edge detection + Sobel gradient magnitude to measure how sharp and dense the structural boundaries are across the sample. Edge orientation variance computed from gradient angles.

🌀

FFT Spectral Features

Power spectrum decomposition: total power, low/high-frequency fraction, dominant spatial frequency, and spectral entropy. Entropy measures how ordered vs. disordered each site's structure is.

⚡

Magnetic Domain Shape (MFM)

Otsu-threshold MFM phase → binary positive/negative domains → label + regionprops for count, mean size, circularity, eccentricity, and a size uniformity index.

🔳

Magnetic Intensity & Contrast

RMS magnetic signal, Michelson contrast, positive vs. negative domain intensity balance, local contrast variation. Tells you how strong and how symmetric the magnetic response is.

How it was built

LLM + physics intuition: the real workflow

This project was built using a hybrid approach: I provided physical understanding and scientific direction, while using an LLM as a coding accelerator to rapidly scaffold boilerplate, debug edge cases, and iterate on visualization layouts. The division of labor looked something like this:

Hour 0–0.5

Data loading & exploration

Unpacked the pickle file, inspected shape (23, 5, 256, 256), visualized raw MFM and height channels to understand data quality, noise level, and scale. Identified channel 0 = height, channel 4 = MFM phase.

Hour 0.5–1.5

Core physics metrics — correlation length & roughness

Implemented FFT autocorrelation for magnetic correlation length. Addressed the TwoSlopeNorm issue on near-zero images. Built plane-leveling and roughness pipeline. Produced first spatial maps.

Hour 1.5–2.5

Particle detection pipeline

Watershed segmentation on height images. Tuned sigma, min_size, and min_distance parameters. Ran across all 23 sites. Detected ~7,165 particles with mean diameter ~18 nm.

Hour 2.5–3+

39-feature extraction suite & comprehensive summary

Added Gabor filters, edge analysis, FFT spectral features, and magnetic domain metrics. Built a cross-feature correlation matrix. Compiled a 4×4 summary figure across all 23 sites with a statistics table.

Code highlight

Magnetic correlation length via Wiener–Khinchin

The most physically interesting metric: how far magnetic order persists across each site. Computed via FFT-based autocorrelation and threshold crossing on the radial average.

Python
# Wiener–Khinchin: autocorrelation from power spectrum
def correlation_length_nm(height_img, pixel_size_nm, smooth_sigma_px=3,
                           thresholds=(0.6, 0.2, 0.1)):
    I = np.asarray(height_img, dtype=float) - np.nanmean(height_img)

    # Hanning window → suppress FFT wrap-around
    wy = np.hanning(I.shape[0])[:, None]
    wx = np.hanning(I.shape[1])[None, :]
    Iw = I * (wy * wx)

    # Autocorrelation via power spectrum
    F  = fft2(Iw)
    C  = fftshift(np.real(ifft2(np.abs(F)**2)))
    Cn = C / C[C.shape[0]//2, C.shape[1]//2]  # normalize at zero-lag

    # Radial average → smooth → find threshold crossing
    r_px, Cr = _radial_profile(Cn)
    Cr = gaussian_filter1d(Cr, sigma=smooth_sigma_px)

    for thr in thresholds:
        idx = np.where(Cr <= thr)[0]
        if idx.size > 0:
            # Linear interpolate for sub-pixel precision
            i = idx[0]
            t = (thr - Cr[i-1]) / (Cr[i] - Cr[i-1] + 1e-12)
            r_star_px = r_px[i-1] + t * (r_px[i] - r_px[i-1])
            return float(r_star_px * pixel_size_nm)

    return float(min(I.shape) * pixel_size_nm)  # FOV-limited fallback

The upper-right cluster shows strongly positive (red-dominant) response, while mixed and near-zero sites appear toward the edges — directly revealing spatial composition gradients in magnetic anisotropy.
Fig. 2 — Frequency-domain (FFT) analysis. Six-panel figure: total spectral power (log scale), low- and high-frequency power fractions, dominant spatial frequency, spectral entropy, and radial power spectra overlay. High-frequency power dominates the center of the sample; spectral entropy is higher where magnetic texture is more disordered. Radial power spectra (lower-center) decay consistently as ~1/f² — characteristic of self-similar magnetic domain textures.

Magnetic correlation length ranged from 1.0 to 9.5 nm across the sample, with a clear spatial gradient — larger domains in the upper-right region, smaller/more disordered domains toward the lower-left. Surface roughness was largely uniform (Ra ~ 2–5 × 10⁻⁹ m) with outlier sites at the edges. Particle density varied ~2× across the library (167 to 488 per site), with size distributions consistently log-normal.

The feature correlation matrix revealed that magnetic RMS signal is largely decoupled from topographic roughness — an important physical result suggesting magnetic ordering is driven by composition rather than surface morphology.

Takeaways

What this taught me

  • 💡Physics knowledge directs LLM output. Knowing why FFT autocorrelation gives correlation length, or why Gabor filters capture texture anisotropy, lets you prompt for exactly the right approach rather than hoping the model guesses correctly.
  • ⚡Modular pipelines scale fast. Writing a reusable create_heatmap() helper early on meant adding new features cost almost no visualization code.
  • 📊Normalization matters enormously. The TwoSlopeNorm issue on near-zero images (MFM at one site was essentially flat) would have silently broken the whole pipeline. Always visualize first.
  • 🌏39 features, 23 sites = real science. A cross-feature correlation matrix isn't just a visualization — it starts to answer materials science questions about which structural properties co-vary and which are independent.
  • ⚒Time-bounded work sharpens priorities. Under 3-hour pressure, you ship the physically meaningful things first and build toward completeness. That's also how high-impact engineering works.

Symbolic Regression for Phase Transitions: Discovering Physics from Monte Carlo Data

📅 March 2026 ⏰ ~3 hours 📉 Ising model + PFM domain switching 🏭 Statistical physics + ML
Context

The challenge

Given Monte Carlo simulation data for the 2D Ising model — specifically the specific heat Cv as a function of exchange coupling J — can an AI discover the underlying physics equation without being told what it is? And separately: given PFM (Piezoresponse Force Microscopy) domain-switching radius data as a function of applied voltage and pulse duration, find the governing equation from data alone.

🧠
Why this matters: The Ising model has a known exact solution (Onsager, 1944), but most real materials systems don't. Symbolic regression — using genetic programming to search for compact mathematical expressions — offers a data-driven path to interpretable physics equations. This is exactly how equation discovery in materials science works in practice.
Approach

PySR: symbolic regression to find equations from data

Used PySR (Python Symbolic Regression), an evolutionary algorithm that searches over mathematical expressions to minimize prediction error subject to a complexity penalty. Multiple candidate equations were generated, ranked by a composite score combining fit quality, physical plausibility, complexity, and extrapolation risk.

📈

Ising model Cv — Monte Carlo data

Specific heat as a function of exchange coupling J, showing the characteristic divergence near the critical point Jc ≈ 1.18. The peak shape encodes finite-size scaling physics.

⚛

Multi-objective equation scoring

Equations scored on: PySR fit score, physics plausibility (peak location and decay behavior), simplicity/complexity, and extrapolation risk beyond the training domain.

🌟

LLM-assisted physics assessment

Each candidate equation was assessed by an LLM for physical interpretability — does it produce a single rounded peak? Does it decay correctly at both limits? Are the functional forms physically motivated?

🔭

PFM domain switching — 2D surface fit

Radius of ferroelectric domain switching as a function of applied voltage U (4–10 V) and pulse duration t (0–2 s). A 3D surface model fitted to the experimental PFM grid data.

Results — Ising model

Candidate equations and their rankings

PySR returned 8 candidate equations of varying complexity. The top-ranked equation (by composite score) achieved R² = 0.9677 on the Monte Carlo Cv data. Six candidates are shown below, color-coded by assessment: GOOD, QUESTIONABLE, and BAD (overfit).

Fig. 3 — All candidate equations vs. Ising model data. Each panel shows a PySR candidate equation (data = gray dots, model = colored line) with rank, complexity c, composite score, and quality label. Rank #1 (orange, QUESTIONABLE) and Rank #4 (green, GOOD) both capture the peak shape well. Rank #6 (red, BAD) overfits with two spurious peaks — a common failure mode of unconstrained symbolic regression at high complexity.
Fig. 4 — Best equation deep-dive. The top-ranked expression (sqrt(Abs(log(Abs(x0)+1) − 0.778)) − 1·0.723)² achieves R² = 0.9677. Residuals (upper right) show systematic structure near the critical point — expected, since the true Onsager singularity is logarithmic and no polynomial/sqrt expression can perfectly capture it. The residual distribution (lower left) is approximately Gaussian. Assessment: QUESTIONABLE due to the nested sqrt(Abs(log(...))) structure — physically functional but not directly interpretable.
Results — PFM domain switching

3D surface model: domain radius vs. voltage & duration

The PFM domain-switching radius increases strongly with applied voltage and weakly with pulse duration. PySR found a compact governing equation that fits the 2D surface well — consistent with known electrostatic models of ferroelectric domain nucleation where radius scales roughly as Uα · tβ with small β.

Fig. 5 — PFM domain switching: best model surface vs. data. Left: 3D surface (model) with experimental data points (white circles). Right: contour map showing radius in nm as a function of voltage U (V) and duration t (s). Radius ranges 60–132 nm. The model shows that voltage is the dominant control parameter — a 2× voltage increase produces ~2× radius increase — while duration contributes a weaker sublinear dependence. This is consistent with thermally-activated nucleation physics.
Takeaways

What this taught me

  • 🧠Symbolic regression finds equations, not just fits. PySR doesn't give you a black-box curve — it gives you a mathematical expression you can examine, interpret, and compare to known physics. That's qualitatively different from fitting a neural network.
  • ⚛Complexity penalty is essential. Without it, the algorithm overfits to noise (see Rank #6: two spurious peaks). The right complexity tradeoff is a physics judgment, not a purely mathematical one.
  • 🔭LLM + domain knowledge = better scoring. The LLM assessment (single peak? correct decay at J→0 and J→∞?) filtered candidates faster than residual analysis alone. Hybrid human-AI scoring is genuinely useful.
  • ⚡The best equation isn't always the top scorer. Rank #1 was "QUESTIONABLE" despite the highest composite score. Rank #4 (tanh-based) was arguably more physically interpretable. Score metrics and physical insight must be used together.
